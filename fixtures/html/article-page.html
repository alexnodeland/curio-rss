Understanding WAL mode

SQLite's write-ahead log changes the concurrency story completely. Instead of writers blocking readers, the WAL lets readers keep a stable snapshot while a single writer appends.

The basics

Enable it with a pragma:

PRAGMA journal_mode = WAL;
PRAGMA synchronous = NORMAL;
-- and always set a busy timeout
PRAGMA busy_timeout = 5000;

Note that <checkpoint> behavior depends on the documentation.

A single writer matches SQLite's actual concurrency model.

Checklist

  • WAL mode on
  • busy_timeout set
  • one writer thread
    • channel-fed
    • transactional
